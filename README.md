# linux_hardening_lab 
# Overview: 
This project documents a hands-on Linux security hardening lab focused on fundamental defensive security practices.  
Topics covered include:  
User and permission management   
Service control  
Secure SSH configuration  
Firewall setup  
Logging configuration  
Basic system security checks  
The goal of this lab is to demonstrate foundational knowledge of Linux system administration and cybersecurity principles by securing a fresh Linux system step by step.  
This lab is intended strictly for educational purposes and reflects defensive best practices commonly used in real-world environments.  

# Lab Environment:  
This lab was performed using:  
Operating System: Kali Linux  
Virtualization Platform: VMware  
Environment Type: Virtual Machine  
The purpose of this project is not offensive security or exploitation, but rather to apply basic Linux hardening principles to a system that contains multiple preinstalled tools and services.  
By securing users, services, network access, and logging, this lab demonstrates defensive security practices applicable to:  
Linux servers  
Enterprise environments  
Cloud virtual machines  
Personal lab environments  

# Key Security Principles Demonstrated:  
Principle of Least Privilege  
Separation of Roles  
Access Control Enforcement  
System Enumeration Awareness  
Secure Configuration Baselines  

# Educational Purpose:  
This lab demonstrates practical defensive security knowledge and foundational Linux administration skills.  
It is designed for:  
Cybersecurity students  
Junior system administrators  
SOC analysts  
Anyone building a home lab for security practice
